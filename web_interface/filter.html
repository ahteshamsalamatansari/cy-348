Filter API Endpoint

Detect contact info sharing and circumvention attempts. The server tracks per-user message history automatically — just send user_id + message.

POST /api/filter

Request Body (JSON):

{
  "message": "The message to analyze",          // required
  "user_id": "unique-user-id",                  // recommended (enables per-user history)
  "username": "display-name",                   // optional alias for user_id
  "context": ["manual", "override", "history"]  // optional (overrides auto-history)
}

Response (JSON):

{
  "safe": true | false,
  "category": "CLEAN" | "PHONE" | "EMAIL" | "SOCIAL" | "CIRCUMVENTION",
  "reason": "Explanation of the detection",
  "user_id": "the-user-id-used",
  "context_size": 3
}

How Per-User History Works:

The server remembers the last 5 messages per user_id.
No client-side history management needed. Just send user_id with each request.
• Sessions auto-expire after 30 minutes of inactivity
• Split phone numbers across messages ARE detected (e.g. "555" then "1234")
• Different user_ids have separate, isolated histories
• If user_id is omitted, messages are tracked under "anonymous" (shared)

Integration Example (Recommended):

// For each user message in your app, call:
const response = await fetch('https://ai.appsscale.com/api/filter', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
        message: userMessage,
        user_id: currentUserId   // your app's user identifier
    })
});
const result = await response.json();
if (!result.safe) {
    // Block or hold the message
    console.log(`Blocked: ${result.category} - ${result.reason}`);
} else {
    // Release message to recipient
    deliverMessage(userMessage);
}

cURL Example:

# First message from user
curl -X POST https://ai.appsscale.com/api/filter \
  -H "Content-Type: application/json" \
  -d '{"message": "my number is 555", "user_id": "driver-123"}'

# Second message — server remembers "555" from before
curl -X POST https://ai.appsscale.com/api/filter \
  -H "Content-Type: application/json" \
  -d '{"message": "1234", "user_id": "driver-123"}'
# → {"safe": false, "category": "PHONE", "reason": "Split phone number (555 + 1234)"}

Python Example:

import requests

def check_message(user_id: str, message: str) -> dict:
    """Check a user message through the filter API."""
    response = requests.post(
        "https://ai.appsscale.com/api/filter",
        json={"message": message, "user_id": user_id}
    )
    return response.json()

# Usage — server tracks history per user automatically
result = check_message("driver-123", "my number is 555")
print(result)  # {"safe": false, "category": "PHONE", ...}

result = check_message("driver-123", "1234")
print(result)  # {"safe": false, "category": "PHONE", "reason": "Split phone (555+1234)"}

Categories:

CLEAN — No contact info detected
PHONE — Phone number detected (including split across messages)
EMAIL — Email address detected (including obfuscated: [at], [dot])
SOCIAL — Social media handle detected
CIRCUMVENTION — Deliberate attempt to bypass filter
